# Manual runtime ownership matrix

- Slice: HELL-036
- Date: 2026-05-29
- Scope: custom Hell browser/runtime behavior that should be kept, deleted from core, or delegated before a production-ready claim.
- Evidence sources: current product paths under `projects/hell/src/lib`, `docs/architecture/target-architecture.md`, `docs/adr/hell-heavy-features.md`, Context7 checks for Angular CDK/Angular Aria and ng-primitives accessibility primitives on 2026-05-29, HELL-040 browser dismissal evidence, HELL-057 Context7 refresh on 2026-06-03, and HELL-058 omnibar focus-dismissal reduction.

## Decision rule

Prefer Angular CDK/A11y, Angular Aria, ng-primitives, or dedicated feature packages when they own focus, keyboard, overlay, grid, worker, or browser-runtime risk better than Hell. Keep Hell-owned runtime only when it is a small documented public contract with tests and a named owner slice.

HELL-048 adds a product architecture check for direct browser globals. Allowed `document`/`window`/`ResizeObserver`/`IntersectionObserver` seams and provisional follow-up owners live in `docs/architecture/browser-global-seams.md` and are enforced by `tools/check-architecture.mjs`.

## Review usage

Architecture reviewers should start from this matrix before flagging manual runtime as a blocker. A finding must name the matrix row, explain whether the row's decision is wrong or untested, and either point to the owner slice above or propose a new narrow slice if the owner is missing.
