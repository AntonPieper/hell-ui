# Manual runtime ownership matrix

- Date: 2026-05-29
- Updated: 2026-07-11
- Scope: custom Hell browser/runtime behavior that should be kept, deleted from
  core, or delegated before a production-ready claim.

## Decision Rule

Prefer Angular CDK/A11y, Angular Aria, ng-primitives, or dedicated feature
packages when they own focus, keyboard, overlay, grid, worker, or browser
runtime risk better than Hell. Keep Hell-owned runtime only when it is a small,
documented public contract with tests.

Allowed direct browser-global seams follow
`docs/architecture/browser-global-seams.md` and are enforced by ESLint
`no-restricted-globals` with justified inline disables.

## Review Usage

Architecture reviewers should start from this matrix before flagging manual
runtime as a blocker. A finding must name the matrix row and explain whether
the row's decision is wrong, untested, or missing a narrow follow-up.
